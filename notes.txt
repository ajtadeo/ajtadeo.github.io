index.html
    when this file is in the root directory, the web browser thinks this is the home page of the website
    good when using one webpage
    HTML defines layouts for a webpage using tags

<!DOCTYPE html>
    tells web browser the type of the document --> html document
    single tag, only needed onces

container tag
    starting tag and ending tag, contains content or other html tags
    EX. <html> ... content ... </html>

head tags
    data about the document
    title, description, other attributes (associated image), import resources
    command center --> meta data about the actual file

body tags
    meat and bones, generally a lot bigger than the head
    content of the webpage, gets parsed and rendered by the browser
    order of documents in html file reflects the order on the website
    dont care about white space, only whats in the tags

meta tag
    meta information about our file

HTML Skeleton:
<!DOCTYPE html>
<html>
    <head>

    </head>
    <body>
        
    </body>
</html>

HEAD TAGS
    <title>Name of Website</title>  // title of website
    <meta charset="UTF-8"> // character set for file
    <meta name="description" content="This is Alyssa's website!"> // set descrition of website

BODY TAGS
    <h1>Header 1</h1> // heading 1-6
    <p>body</p> // good for writing blocks of text, format themselves
    <b>text</b> // bolds certain words in a body of text
    <i>text</i> // italicizes text
    <br/> // break tag, new line
    <hr/> // horizontal rule, line across website
    <big>text</big> // makes text bigger
    <small>text</small> // makes text smaller
    <sub>text</sub> // subscript text
    <sup>text</sup> // superscript text
